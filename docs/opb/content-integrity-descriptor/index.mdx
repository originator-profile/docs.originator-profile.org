# Content Integrity Descriptor

Content Integrity Descriptor はコンテンツ内の特定の情報の完全性を保証するための仕組みです。 Content Integrity Descriptor としてさまざまな種類を定義することができます。個々の Content Integrity Descriptor は対象の情報の位置を特定する方法、情報を取得する方法、その情報の完全性を検証する方法を持ち、完全性を保証したい情報に合わせて使い分けたり、新たに定義することができます。

## 登録プロセス

新しい Content Integrity Descriptor を作る場合、少なくとも次のことを定義してください (MUST):

- タイプ名
- プロパティ定義
- HTML への適用方法
- 検証プロセス
- 要素位置特定方法

また、タイプ名の衝突を避けるため、定義した Content Integrity Descriptor は [Content Integrity Type Registry](#registry) に登録してください (MUST)。

## Content Integrity Type Registry \{#registry}

- `HTMLTargetIntegrity`: [HTML Fragment Integrity](./html.md)
- `TextTargetIntegrity`: [Text within DOM Integrity](./text.md)
- `VisibleTextTargetIntegrity`: [Visible Text within DOM Integrity](./visible-text.md)
- `ExternalResourceTargetIntegrity`: [External Resource Integrity](./external-resource.md)
